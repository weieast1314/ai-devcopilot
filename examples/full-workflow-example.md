# 使用示例：标准开发流、热修复与中断恢复

本文档展示如何使用 `AI DevCopilot` 完成三类典型场景：
1. 标准需求开发
2. 热修复
3. 会话中断后的继续执行

## 场景二：热修复

问题：线上登录 token 在 30 分钟后异常过期，需要立即修复。

### 1. 发起热修复

```text
用户: 线上问题需要紧急修复，登录 token 30 分钟后会异常过期

AI 编辑器:
已识别为热修复请求，正在进入 hotfix-flow。

[1/3] 问题定位
      ✓ 影响范围: 登录鉴权 / token 刷新逻辑
      ✓ 初步根因: token 过期时间校验边界错误

[2/3] 修复摘要
      ✓ 修复目标: 调整过期判断逻辑
      ✓ 最小步骤: 修复校验 → 补回归验证
      ✓ 执行边界: 仅处理当前故障，不做顺手重构

[3/3] 当前状态
      ⏸️ 等待用户回复：确认修复，开始执行
```

### 2. 确认后执行

```text

AI 编辑器:
```

## 场景三：会话中断后的继续执行

场景：你在执行到一半时关闭了编辑器，重新打开后想继续。

### 推荐恢复说法

```text
用户: 继续刚才的任务：当前分支 feat/23182-ai-call-log，计划已确认，已经完成实体和 Mapper，继续执行 Service、SQL、验证和交付
```

### AI 的理想响应

```text
AI 编辑器:
已恢复当前上下文。

当前阶段：代码实现
- 已完成：2/4
- 已完成项：AiCallLogDO、AiCallLogMapper
- 待完成项：AIModelServiceImpl、SQL 脚本、验证、交付
- 下一步：继续执行第 3 项任务
```

---

## 推荐用法总结

- 要开始：优先说“开始开发”或直接贴需求
- 要先出方案：说“进入计划模式”
- 要开始改代码：说“确认计划，开始执行”
- 要验证：说“验证代码”
- 要提交推送部署：说“代码交付”
- 要热修复：说“热修复”或说明“线上问题需要紧急修复”
