# Dev Flow (标准开发流程)

这是一个完整的端到端开发流程 Pipeline，包含从需求获取到代码交付的所有阶段。

## 协作约束

在整个 Dev Flow 中，必须遵循以下强约束：

1. **强制 superpowers**：所有阶段必须使用 superpowers 过程型 Skills，缺失时流程阻断。
2. **禁止降级**：任何辅助能力（交互确认、状态持久化等）缺失时均阻断，不提供降级方案。
3. **先计划，后执行**：进入实现阶段前，必须先完成计划文档。
4. **计划待确认**：计划生成后默认暂停，等待用户明确回复"确认计划，开始执行"后才能改代码。
5. **只按当前计划执行**：实现阶段仅允许执行当前计划中的任务，不得擅自扩大范围。
6. **偏差先更新计划**：如果发现遗漏、依赖变化或方案偏差，先同步更新计划，再继续执行。
7. **完成一项，汇报一项**：每完成一个任务，都要汇报任务状态、修改文件、验证结果和下一步。
8. **阶段切换有提示**：每个阶段结束时，使用 `ask_followup_question` 工具提供结构化选择，引导下一步动作。
9. **流程状态持久化**：每个阶段完成后保存状态到 `.ai-devcopilot/state/flow-state.json`。

## 阶段详情

### 阶段 1: 需求获取

**目标**: 从不同来源获取并标准化需求信息

| 步骤 | Skill | 说明 |
|------|-------|------|
| 1.1 | `requirement-fetch` | 多来源需求获取，输出标准化需求 |

**输出**:
- 标准化的需求信息
- 建议的分支名称

```
--- 📋 阶段 1/4: 需求获取 ---

[requirement-fetch] 执行中...

✓ 来源: 飞书文档
✓ 标题: 用户登录
✓ Issue: #23181
✓ 类型: feat
✓ 分支: feat/23181-user-login

┌─────────────────────────────────────────┐
│  📊 整体进度: ████░░░░░░░░░░░░ 25%      │
│     阶段 1 ✓ → 阶段 2 → 阶段 3 → 阶段 4  │
└─────────────────────────────────────────┘

✅ 当前阶段已完成！
```

> **阶段引导**: 完成后必须使用 `ask_followup_question` 工具：
> ```
> 问题: "需求信息已获取并标准化，请选择下一步操作："
> 选项:
>   1. "继续 - 创建开发分支并生成执行计划（进入下一阶段）" (trigger: "继续")
>   2. "修改需求信息 - 调整标题、类型或分支名称建议" (trigger: "修改需求")
>   3. "取消流程 - 放弃当前需求处理" (trigger: "取消")
> ```

**输出**:
- 开发分支
- 执行计划文档

```

[requirement-to-branch]
✓ 分支: feat/23181-ai-classification

[writing-plans]
✓ 计划: 计划文档已生成（由编辑器自动管理）
✓ 任务数: 5
✓ 待确认: 技术方案 / 任务顺序 / 影响范围

┌─────────────────────────────────────────┐
│  📊 整体进度: ████████░░░░░░░░ 50%      │
│     阶段 1 ✓ → 阶段 2 ✓ → 阶段 3 → 阶段 4 │
└─────────────────────────────────────────┘

⏸️  当前状态: 已暂停，等待用户确认计划
```

> **阶段引导**: 计划生成后必须使用 `ask_followup_question` 工具：
> ```
> 问题: "执行计划已生成，包含任务清单、执行顺序和风险点。请确认下一步操作："
> 选项:
>   1. "确认计划，开始执行 - 按计划逐项执行代码修改" (trigger: "确认计划，开始执行")

> **阶段引导**: 实现完成后必须使用 `ask_followup_question` 工具：
> ```

| 4.2 | `code-delivery` | 提交+推送+部署 |

## 流程控制

### 能力依赖（强制）

Dev Flow 依赖以下能力，缺失任何一项都将阻断流程：

| 能力 | 用途 | 缺失行为 |
|------|------|----------|
| `skill.superpowers.available` | 会话级能力预检 | 阻断 |
| `skill.superpowers.brainstorming` | 计划阶段 brainstorming | 阻断 |
| `skill.superpowers.writing-plans` | 计划阶段 plan 生成 | 阻断 |
| `interaction.ask_followup` | 结构化选项确认 | 阻断 |
| `state.persistence` | 流程状态持久化 | 阻断 |
| `skill.superpowers.executing-plans` | 实现阶段执行 | 阻断 |
| `shell.exec` | 命令执行/验证 | 阻断 |
| `skill.superpowers.verification-before-completion` | 验证阶段 | 阻断 |
| `skill.superpowers.code-review` | 代码审查 | 阻断 |
| `skill.superpowers.finishing-branch` | 分支收尾交付 | 阻断 |

### 中断与恢复

- 阶段 2 完成后必须暂停，等待用户确认计划
- 阶段 3 执行过程中支持从中断点恢复执行
- 保存中间状态到 `.ai-devcopilot/state/`

### 执行约束

- 只允许执行当前计划中的任务
- 若出现计划外需求或技术偏差，必须先更新计划
- 每完成一个任务都要同步任务状态并向用户汇报
- **禁止降级**：任何辅助能力缺失均阻断，不提供降级方案

### 错误处理

- 阶段内错误：暂停并提示解决方案
- 跨阶段错误：记录状态，支持重试
- 遇到阻塞项：明确说明阻塞原因、影响任务和可选处理方案
- **所有降级路径已禁用**：遇到能力缺失时必须启用相应能力后才能继续
