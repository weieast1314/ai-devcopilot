# Code Review (代码审查)

这个技能用于在提交前对代码变更进行审查，确保代码质量和安全性。

## 输入标准化

- **输入**:
  - 当前分支的所有变更文件
  - 或指定的文件列表
- **可选输入**: 审查重点（安全/性能/规范）

## 输出标准化

- **格式**: 审查报告
- **内容**:
  - 发现的问题列表
  - 严重程度（高/中/低）
  - 修复建议
  - 通过/需修改

## 执行流程

### 步骤 1: 获取变更

```bash
git diff --name-only
git diff
```

### 步骤 2: 分类审查

```
--- 🔍 代码审查 ---

审查范围: feat/23182-ai-call-log
变更文件: 5 个

[1/3] 安全检查
      ⚠ 中等: AiModelServiceImpl.java:45
            日志中包含模型响应内容，可能泄露敏感数据
      
📊 进度: 1/3
```

### 步骤 3: 生成报告

```
[2/3] 代码规范
      ✓ 无问题

[3/3] 性能检查
      ⚠ 低: AiCallLogMapper.xml:23
            INSERT 语句未使用批量插入

📊 状态: 审查完成

总结: 2 个问题需关注，建议修复后提交
```

## 审查清单

### 安全检查
- [ ] 无硬编码密码/API 密钥
- [ ] SQL 使用参数化查询
- [ ] 用户输入已验证和转义
- [ ] 敏感数据已加密

### 代码规范
- [ ] 类/方法命名清晰
- [ ] 关键逻辑有注释
- [ ] 无重复代码

### 性能检查
- [ ] 无 N+1 查询问题
- [ ] 合理使用缓存

## 使用示例

**输入**:
```
审查当前分支的代码变更
```

**输出**:
```
审查范围: feat/23182-ai-call-log
变更文件: 5 个

[1/3] 安全检查 ⚠ 1 个问题
[2/3] 代码规范 ✓
[3/3] 性能检查 ⚠ 1 个问题

总结: 2 个问题需关注
```
